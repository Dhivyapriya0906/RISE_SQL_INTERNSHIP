# RISE_SQL_INTERNSHIP
# Database Projects (SQL / MongoDB)

This repository contains database design projects developed as part of the **RISE Internship – Database (SQL / MongoDB)** program. The goal of these projects is to practice real-world database design, relational modeling, and SQL query implementation for different business domains.

---

## Tools Used

* MySQL
* SQL
* MongoDB (conceptual design)
* Database normalization techniques

---

# Project 1 – Business Application Database System

This project focuses on designing a relational database for a general business application. The system manages customers, products, orders, and order details.

### Features

* Customer and product management
* Order and transaction handling
* Relationship management using foreign keys
* CRUD operations (Create, Read, Update, Delete)
* SQL joins and aggregate queries

### Tables

Customer, Product, Orders, OrderDetails

---

# Project 2 – E-Commerce Database System

This project simulates an e-commerce platform database that manages products, users, orders, and payment information.

### Features

* Product catalog management
* User and order management
* Transaction and payment tracking
* Complex SQL joins and queries
* Database indexing and normalization

### Tables

Users, Products, Orders, OrderItems, Payments

---

# Project 3 – Banking and Financial Transaction Database System

This project implements a secure banking database that manages customer accounts and financial transactions.

### Features

* Customer account management
* Deposit and withdrawal transaction tracking
* Transaction consistency and integrity
* SQL transaction control (commit / rollback)
* Aggregate queries for financial analysis

### Tables

Customers, Accounts, Transactions

---

# Project 4 – Healthcare Management Database System

This project models a hospital database used to manage patient records, doctor information, appointments, and billing.

### Features

* Patient record management
* Doctor and specialization information
* Appointment scheduling system
* Billing and payment tracking
* Reporting queries using SQL joins

### Tables

Patients, Doctors, Appointments, Billing

---

## Learning Outcomes

Through these projects I learned:

* Relational database schema design
* Data normalization
* SQL CRUD operations
* Complex joins and aggregation queries
* Real-world database modeling
